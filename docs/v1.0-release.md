# v1.0 Release Notes

v1.0 adds market-based allocation, cross-Hive federation, an optimiser,
enterprise RBAC/audit, and multi-node clustering.

## Migration

`006_v10_features.sql` (SQLite) and the parallel `postgres/006_v10_features.sql`
add 7 new tables (`auctions`, `bids`, `agent_tokens`, `federation_links`,
`audit_log`, `rbac_users`, `cluster_members`) plus a `tenant_id` column on
`agents`, `tasks`, and `workflows` (default `'default'` so existing rows stay valid).

## New features

### Market allocation (Stories 18.1-18.3)

```bash
hive auction open <task-id> --strategy lowest-cost    # or fastest, best-reputation
hive auction close <auction-id>
hive wallet balance <agent>
hive wallet credit <agent> <amount>
```

`market.Store` persists auctions and bids; `market.NewAuction(nil).SelectWinner(bids, strategy)`
picks a winner. Token wallet debits never go below zero.

### Federation + mTLS (Stories 19.1-19.3)

```bash
hive federation add <name> <url> \
  --shared code-review,translate \
  --ca ca.pem --cert client.pem --key client.key
hive federation list
hive federation remove <name>
```

`federation.Store.TLSConfigFor(name)` builds a `*tls.Config` from the stored
PEMs; `BuildClient()` returns an mTLS-ready `*http.Client`.

### Optimiser (Stories 20.1-20.3)

```bash
hive optimize                 # ad-hoc recommendations
hive optimize --trend         # current vs previous 7-day window stats
hive optimize --auto-tune     # suggested config tunings
hive optimize --json          # machine-readable output
```

`AutoTune` reads `Trend()` and proposes lowering the breaker threshold when
failure rate regresses or lengthening retry backoff when latency spikes.

### Enterprise (Stories 21.1-21.4)

- RBAC user store: `hive users add <subject> <role> [--tenant]` / `hive users list`
- Audit log: `hive audit list [--since 24h] [--format table|json|csv]`
- Audit export: `hive audit export <path>` (JSON or CSV by extension)
- Tenant column on core tables (default `'default'`)

Roles: `admin`, `operator`, `viewer` (existing `auth.RBACMiddleware` enforces them).

### Multi-node (Stories 22.1-22.3)

- **PostgreSQL storage backend**: set `storage: postgres` and
  `postgres_url: postgres://…` in `hive.yaml` (or `HIVE_STORAGE` /
  `HIVE_POSTGRES_URL` env vars). A parallel migration set in
  `internal/storage/migrations/postgres/` runs at startup.
- **Cluster roster**: `cluster.Roster` persists node heartbeats with a
  `MarkStale(maxAge)` sweeper for offline detection.
- **Node-aware routing**: `Manager.PickAgent(perNode, taskType)` implements
  `local-first` (preferred when `routing: local-first` or unset) vs `best-fit`
  (deterministic round-robin across sorted node IDs).

## Breaking changes

None. v0.3 databases upgrade in place.
