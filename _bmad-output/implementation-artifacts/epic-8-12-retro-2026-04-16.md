## What Went Well

### 1. Single-Binary Dashboard Works Beautifully
Svelte 5 with static adapter builds to `internal/dashboard/dist/`, Go embeds it via `//go:embed`, and `hive serve` serves everything on one port. Zero separate frontend deployment. The developer experience is exactly what we planned.

### 2. Trust Engine Is Clean and Testable
The graduated autonomy engine (4 levels, auto-promotion, manual override, never auto-demote) is well-isolated in its own package with 6 tests. The threshold system is configurable and the trust history table provides full audit trail.

### 3. Code Review Found Real Security Issues Again
The adversarial review caught: WebSocket race condition (map mutation under RLock), SSRF via webhook URLs, WebSocket origin bypass. All fixed. This validates running code review on every release.

### 4. Knowledge Layer Keyword Search Works Without External Dependencies
Simple TF-IDF-like keyword matching with recency decay — no vector database, no embeddings API, pure Go. Good enough for MVP. Upgrade path to real embeddings is clear for v0.3.

## What We Learned

### 1. SPA Routing Needs Server-Side Fallback
The 404 bug on direct URL navigation (`/events`, `/agents`) was caught only by testing in the browser. The Go file server doesn't know about SvelteKit routes. **Always test SPA apps with direct URL access, not just client-side navigation.**

### 2. SSRF Prevention Breaks Test Patterns
Adding webhook URL validation broke tests using `httptest.NewServer` (localhost URLs). Tests had to insert directly into the database to bypass validation. **Consider adding a `skipValidation` flag for testing, or use an allowlist approach that includes test URLs.**

### 3. WebSocket Concurrency Is Tricky
gorilla/websocket doesn't support concurrent writes to the same connection. The initial implementation used RLock for broadcast (allowing concurrent broadcasts) which is unsafe. **Per-connection write mutexes are essential.**

### 4. Embed Errors Should Panic, Not Silently Fail
The `fs.Sub` error was silently ignored, meaning if assets weren't embedded properly, the server would start but serve nothing. **Embed failures should panic at init time with a clear message.**
